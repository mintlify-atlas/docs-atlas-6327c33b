## Prerequisites Installation

<Steps>
  <Step title="Install Java 17">
    Download and install Java 17 from [Eclipse Adoptium](https://adoptium.net/):

    <CodeGroup>

    ```bash Ubuntu/Debian
    # Add Adoptium repository
    wget -O - https://packages.adoptium.net/artifactory/api/gpg/key/public | sudo apt-key add -
    echo "deb https://packages.adoptium.net/artifactory/deb $(awk -F= '/^VERSION_CODENAME/{print$2}' /etc/os-release) main" | sudo tee /etc/apt/sources.list.d/adoptium.list

    # Install Java 17
    sudo apt update
    sudo apt install temurin-17-jdk

    # Verify installation
    java -version
    ```

    ```bash macOS
    # Using Homebrew
    brew install openjdk@17

    # Add to PATH
    echo 'export PATH="/opt/homebrew/opt/openjdk@17/bin:$PATH"' >> ~/.zshrc
    source ~/.zshrc

    # Verify installation
    java -version
    ```

    ```bash Windows
    # Download installer from https://adoptium.net/
    # Run the MSI installer
    # Add JAVA_HOME to environment variables:
    # JAVA_HOME=C:\Program Files\Eclipse Adoptium\jdk-17.x.x

    # Verify installation
    java -version
    ```

    </CodeGroup>

    Expected output:
    ```
    openjdk version "17.0.x" 2024-xx-xx
    OpenJDK Runtime Environment Temurin-17.0.x
    ```
  </Step>

  <Step title="Install Maven">
    Download and install Apache Maven:

    <CodeGroup>

    ```bash Ubuntu/Debian
    sudo apt update
    sudo apt install maven

    # Verify installation
    mvn -version
    ```

    ```bash macOS
    brew install maven

    # Verify installation
    mvn -version
    ```

    ```bash Manual Installation (All Platforms)
    # Download Maven
    wget https://dlcdn.apache.org/maven/maven-3/3.9.6/binaries/apache-maven-3.9.6-bin.tar.gz

    # Extract
    tar -xvf apache-maven-3.9.6-bin.tar.gz
    sudo mv apache-maven-3.9.6 /opt/

    # Add to PATH
    echo 'export PATH=/opt/apache-maven-3.9.6/bin:$PATH' >> ~/.bashrc
    source ~/.bashrc

    # Verify
    mvn -version
    ```

    </CodeGroup>
  </Step>

  <Step title="Install MySQL">
    Install MySQL database server:

    <CodeGroup>

    ```bash Ubuntu/Debian
    sudo apt update
    sudo apt install mysql-server

    # Start MySQL service
    sudo systemctl start mysql
    sudo systemctl enable mysql

    # Secure installation
    sudo mysql_secure_installation
    ```

    ```bash macOS
    brew install mysql

    # Start MySQL service
    brew services start mysql

    # Secure installation
    mysql_secure_installation
    ```

    ```bash Docker Alternative
    # Run MySQL in a container
    docker run -d \
      --name mysql-drivex \
      -e MYSQL_ROOT_PASSWORD=your_password \
      -e MYSQL_DATABASE=drivex \
      -p 3306:3306 \
      mysql:8.0
    ```

    </CodeGroup>
  </Step>

  <Step title="Configure MySQL Database">
    Create the database and user for DriveX:

    ```sql
    # Connect to MySQL
    mysql -u root -p

    # Create database
    CREATE DATABASE drivex CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

    # Create user (optional, for security)
    CREATE USER 'drivex_user'@'localhost' IDENTIFIED BY 'secure_password';

    # Grant privileges
    GRANT ALL PRIVILEGES ON drivex.* TO 'drivex_user'@'localhost';
    FLUSH PRIVILEGES;

    # Verify database creation
    SHOW DATABASES;
    USE drivex;
    ```

    <Warning>
      Replace `secure_password` with a strong password. Never use default passwords in production.
    </Warning>
  </Step>
</Steps>

## Application Installation

<Steps>
  <Step title="Clone the Repository">
    Clone the DriveX Backend repository:

    ```bash
    git clone https://github.com/rofaba/drivex-backend.git
    cd drivex-backend
    ```
  </Step>

  <Step title="Configure Application Properties">
    Edit `src/main/resources/application.properties` with your environment settings:

    ```properties
    # Application Name
    spring.application.name=DriveX-backend

    # Database Configuration
    spring.datasource.url=jdbc:mysql://localhost:3306/drivex
    spring.datasource.username=drivex_user
    spring.datasource.password=secure_password
    spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

    # JPA/Hibernate Configuration
    spring.jpa.hibernate.ddl-auto=update
    spring.jpa.show-sql=true
    spring.jpa.properties.hibernate.format_sql=true
    spring.jpa.database-platform=org.hibernate.dialect.MySQLDialect

    # Server Configuration
    server.servlet.context-path=/api
    server.port=${PORT:8080}
    server.address=0.0.0.0
    ```

    <Note>
      The `${PORT:8080}` syntax means the server will use the `PORT` environment variable if set, otherwise default to 8080.
    </Note>

    ### Configuration Options Explained

    | Property | Description | Default |
    |----------|-------------|----------|
    | `spring.datasource.url` | JDBC connection URL to MySQL | `jdbc:mysql://localhost:3306/drivex` |
    | `spring.jpa.hibernate.ddl-auto` | Database schema generation strategy | `update` |
    | `spring.jpa.show-sql` | Log SQL queries to console | `true` |
    | `server.servlet.context-path` | Base path for all endpoints | `/api` |
    | `server.port` | HTTP server port | `8080` |
    | `server.address` | Server bind address | `0.0.0.0` |

    <Warning>
      For production, set `spring.jpa.show-sql=false` to avoid logging sensitive data.
    </Warning>
  </Step>

  <Step title="Build the Application">
    Compile and package the application:

    ```bash
    # Clean build with tests
    mvn clean install

    # Or skip tests for faster build
    mvn clean install -DskipTests
    ```

    This generates the JAR file: `target/DriveX-0.0.1-SNAPSHOT.jar`

    <Note>
      The build process includes:
      - Dependency resolution from `pom.xml`
      - Java compilation with annotation processing (Lombok)
      - Resource bundling
      - JAR packaging with embedded Tomcat
    </Note>
  </Step>

  <Step title="Run the Application">
    Start the application using one of these methods:

    <CodeGroup>

    ```bash Maven Spring Boot Plugin
    mvn spring-boot:run
    ```

    ```bash JAR Execution
    java -jar target/DriveX-0.0.1-SNAPSHOT.jar
    ```

    ```bash With Custom Port
    PORT=8081 java -jar target/DriveX-0.0.1-SNAPSHOT.jar
    ```

    ```bash With External Config
    java -jar target/DriveX-0.0.1-SNAPSHOT.jar \
      --spring.config.location=file:./config/application.properties
    ```

    </CodeGroup>
  </Step>

  <Step title="Verify Installation">
    Test the API to ensure it's running correctly:

    ```bash
    # Check server health
    curl http://localhost:8080/api/vehicles

    # Expected response: [] or list of vehicles
    ```

    You should see logs indicating successful startup:

    ```
    2026-03-01 10:00:00.000  INFO --- [main] c.D.D.DriveXBackendApplication
    Started DriveXBackendApplication in 3.456 seconds (JVM running for 4.123)
    ```
  </Step>
</Steps>

## Production Deployment

### Using Systemd Service (Linux)

Create a systemd service for automatic startup and management:

<Steps>
  <Step title="Create Service User">
    ```bash
    sudo useradd -r -s /bin/false drivex
    sudo mkdir -p /opt/drivex
    sudo cp target/DriveX-0.0.1-SNAPSHOT.jar /opt/drivex/app.jar
    sudo chown -R drivex:drivex /opt/drivex
    ```
  </Step>

  <Step title="Create Service File">
    Create `/etc/systemd/system/drivex.service`:

    ```ini
    [Unit]
    Description=DriveX Backend API
    After=syslog.target network.target mysql.service

    [Service]
    User=drivex
    Group=drivex
    WorkingDirectory=/opt/drivex
    ExecStart=/usr/bin/java -jar /opt/drivex/app.jar
    SuccessExitStatus=143
    StandardOutput=journal
    StandardError=journal
    Restart=always
    RestartSec=10

    [Install]
    WantedBy=multi-user.target
    ```
  </Step>

  <Step title="Enable and Start Service">
    ```bash
    sudo systemctl daemon-reload
    sudo systemctl enable drivex
    sudo systemctl start drivex

    # Check status
    sudo systemctl status drivex

    # View logs
    sudo journalctl -u drivex -f
    ```
  </Step>
</Steps>

### Docker Deployment

For containerized deployment:

<Steps>
  <Step title="Build Docker Image">
    The project includes a multi-stage Dockerfile:

    ```dockerfile
    # Etapa 1: Build with Maven
    FROM maven:3.9-eclipse-temurin-17-alpine AS build
    WORKDIR /app
    COPY pom.xml .
    COPY src ./src
    RUN mvn -DskipTests clean package

    # Etapa 2: Runtime image
    FROM eclipse-temurin:17-jdk
    WORKDIR /app
    COPY --from=build /app/target/DriveX-0.0.1-SNAPSHOT.jar app.jar
    EXPOSE 8080
    ENTRYPOINT ["java", "-jar", "app.jar"]
    ```

    Build the image:

    ```bash
    docker build -t drivex-backend:latest .
    ```
  </Step>

  <Step title="Run with Docker Compose">
    Create `docker-compose.yml`:

    ```yaml
    version: '3.8'

    services:
      mysql:
        image: mysql:8.0
        environment:
          MYSQL_ROOT_PASSWORD: rootpassword
          MYSQL_DATABASE: drivex
          MYSQL_USER: drivex_user
          MYSQL_PASSWORD: secure_password
        ports:
          - "3306:3306"
        volumes:
          - mysql-data:/var/lib/mysql
        healthcheck:
          test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
          interval: 10s
          timeout: 5s
          retries: 5

      api:
        image: drivex-backend:latest
        depends_on:
          mysql:
            condition: service_healthy
        environment:
          SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/drivex
          SPRING_DATASOURCE_USERNAME: drivex_user
          SPRING_DATASOURCE_PASSWORD: secure_password
        ports:
          - "8080:8080"
        restart: unless-stopped

    volumes:
      mysql-data:
    ```

    Start services:

    ```bash
    docker-compose up -d

    # View logs
    docker-compose logs -f api
    ```
  </Step>
</Steps>

## Environment Variables

Override configuration using environment variables:

| Variable | Description | Example |
|----------|-------------|----------|
| `PORT` | Server port | `8080` |
| `SPRING_DATASOURCE_URL` | Database connection URL | `jdbc:mysql://localhost:3306/drivex` |
| `SPRING_DATASOURCE_USERNAME` | Database username | `drivex_user` |
| `SPRING_DATASOURCE_PASSWORD` | Database password | `secure_password` |
| `SPRING_JPA_HIBERNATE_DDL_AUTO` | Schema generation | `update`, `validate`, `none` |
| `SPRING_JPA_SHOW_SQL` | Log SQL queries | `true`, `false` |

## Security Considerations

<Warning>
  Before deploying to production, review these security measures:
</Warning>

1. **Change Default Passwords**: Never use default or simple passwords
2. **Enable HTTPS**: Use a reverse proxy (Nginx, Apache) with SSL/TLS certificates
3. **Firewall Rules**: Restrict MySQL port (3306) to localhost only
4. **Environment Variables**: Store sensitive data in environment variables, not in `application.properties`
5. **SQL Injection**: The JPA layer provides protection, but always validate user input
6. **Update Dependencies**: Regularly update Spring Boot and dependencies for security patches

## Troubleshooting

<AccordionGroup>
  <Accordion title="Application fails to start - Port already in use">
    Change the port or stop the conflicting process:
    ```bash
    # Find process using port 8080
    lsof -i :8080
    # Or on Windows
    netstat -ano | findstr :8080

    # Kill the process or change port
    PORT=8081 java -jar target/DriveX-0.0.1-SNAPSHOT.jar
    ```
  </Accordion>

  <Accordion title="Cannot connect to MySQL - Access denied">
    Verify MySQL credentials and permissions:
    ```sql
    # Connect as root
    mysql -u root -p

    # Check user exists
    SELECT User, Host FROM mysql.user WHERE User='drivex_user';

    # Re-grant permissions if needed
    GRANT ALL PRIVILEGES ON drivex.* TO 'drivex_user'@'localhost';
    FLUSH PRIVILEGES;
    ```
  </Accordion>

  <Accordion title="Hibernate schema validation fails">
    If you get schema validation errors:
    
    1. Drop and recreate database:
    ```sql
    DROP DATABASE drivex;
    CREATE DATABASE drivex CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
    ```

    2. Or change `ddl-auto` to recreate tables:
    ```properties
    spring.jpa.hibernate.ddl-auto=create-drop
    ```

    <Warning>
      `create-drop` will delete all data on restart. Use only in development!
    </Warning>
  </Accordion>

  <Accordion title="Maven build fails - dependency resolution">
    Clear Maven cache and retry:
    ```bash
    # Clear local repository cache
    rm -rf ~/.m2/repository

    # Force update dependencies
    mvn clean install -U
    ```
  </Accordion>

  <Accordion title="Lombok compilation errors">
    Ensure Lombok annotation processor is configured in `pom.xml` (line 87-93):
    ```xml
    <annotationProcessorPaths>
        <path>
            <groupId>org.projectlombok</groupId>
            <artifactId>lombok</artifactId>
            <version>1.18.32</version>
        </path>
    </annotationProcessorPaths>
    ```

    If using an IDE, install the Lombok plugin:
    - IntelliJ IDEA: Settings → Plugins → Search "Lombok"
    - Eclipse: Download `lombok.jar` and run it to install
  </Accordion>
</AccordionGroup>
